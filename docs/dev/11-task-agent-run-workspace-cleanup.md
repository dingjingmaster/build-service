# Agent Run 工作区清理任务记录

> 日期：2026-06-26
> 级别：L2
> 状态：已完成

## 背景

用户要求 agent 编译目录不要长期累积：脚本执行成功后自动删除本次工作区；脚本失败时保留现场；agent 每次启动时自动清理历史编译工作区。

## 实现

- agent 启动时清理 `<work_dir>/runs` 下所有 `run_*` 条目。
- 单次 run 的脚本退出码为 `0` 后，agent 自动删除当前 `<work_dir>/runs/<run_id>`。
- 脚本非 0、超时、取消或 agent 执行异常时不自动删除当前工作区，保留用于排查。
- 清理范围限制为 `run_*` 前缀，避免误删 `runs` 目录下其他手工内容。
